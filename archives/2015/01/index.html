2015

Jan 16 2015

hexo，终于搞出了第一篇日志。

从2015年01月07日00点30开始搞hexo，弄了一晚上，非IT男，难得的劣势体现出来了。多亏L出手搭救。。。总结下：

1、安装了Node.js，mysygit，Sbulime Text。

2、在我自己这边的盘内（G:\ABC）右键后点击（git bash）。

键入：

hexo init
npm install

3、用Sbulime Text打开当前路径（G:\ABC）中的（_config.yml），在最后的deploy项下编辑：

deploy:
type: github
repository: https://github.com/math6/math6.github.git
branch: master

之后用git bash完成部署：

hexo generate
hexo deploy

其实这两个命令可以合并起来 hexo g && hexo d

4、创建密钥。私钥创建：

ssh-keygen -t rsa -C “41010@qq.com” -f ~/.ssh/github

我没回车输密码。
然后在（C:）中找到了（.ssh）文件目录。
用记事本或者Sublime打开github.pub，复制出里边的密钥，并粘贴到github账户管理中的SSH KEY内。
也可以用notepad命令打开写字板。

因为一直密钥失败，所以找了原因，应该是使用了之前生成过的密钥，但是后来因为重复操作又生成了新的密钥，所以冲突了。

5、获取网站的地址：

在github官方主页上登录，将公钥的.com改为.io。
这样最终的浏览地址就改为https://math6.github.io。
注意，这边一定要加“s”。

然后在本地做测试。先在路径（G:\ABC）中右键点击（git bash），然后键入：

hexo s

千万不要关闭，这是本地调试要求。然后在浏览器处键入：

http://localhost:4000/

本地调试完毕后，CTRL+C终止hexo s。
内容无误后，hexo g && d。
上传成功。

PS：感谢L同学大力支持。
